We saw this on the last page, but when a button is clicked it calls the onclick handler. More generally, when something happens (to almost any element), it creates an event. We can tie a function (called a handler) to the event, and do things when the event happens. In these examples, we will have some buttons, and have the handlers change things on the web page. Be sure to look at the HTML and JavaScript to see what is going on. There is a lot of discussion in the comments

An obvious, but important thing here is that our code only gets run when an event happens. Otherwise, we don't do anything. This is called event-driven programming.

Different kinds of HTML elements generate different kinds of events.
